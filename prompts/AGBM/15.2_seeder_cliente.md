3. Reseeding: Incluye DBCC CHECKIDENT ('dbo.Clientes', RESEED, 30) al final para sincronizar el contador.
4. Integridad Transaccional: Envuelve todo el proceso en un bloque BEGIN TRANSACTION / COMMIT con manejo de errores mediante TRY...CATCH.
5. Validación: Incluir un comando final SELECT COUNT(*) FROM [dbo].[Clientes] para confirmar la carga de los 30 registros.
</sql_best_practices>

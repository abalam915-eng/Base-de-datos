<technical_requirements>
  <rule_set name="Expansión del Menú Principal">
    - PRESERVAR: Todas las opciones existentes (Gestión de Inventario, Gestión de Clientes, Salir) deben permanecer sin cambios.
    - AGREGAR: Una nueva opción (número 3 o reajustar el orden): "Consultar Historial de Ventas por Fecha".
    - Mantener "Salir" como la última opción disponible.
    - La estructura de scopes y resolución de servicios existente debe permanecer intacta.
  </rule_set>
  
  <rule_set name="Captura de Rango de Fechas">
    - SOLICITAR: Dos valores de tipo DateTime:
      * "Fecha de Inicio (formato: yyyy-MM-dd)".
      * "Fecha de Fin (formato: yyyy-MM-dd)".
    - VALIDACIÓN: Usar `DateTime.TryParse()` con formato invariante o específico para validar las entradas.
    - REINTENTOS: Si la entrada es inválida, mostrar un mensaje de error y permitir reintentar (máximo 3 intentos recomendado).
    - LÓGICA: Validar que StartDate <= EndDate. Si no se cumple, informar al usuario y permitir re-ingresar.
  </rule_set>
  
  <rule_set name="Lógica de Consulta de Ventas por Rango">
    - CREAR: Un objeto SaleFilter con las fechas capturadas.
    - CONSUMIR: El caso de uso IFetchSalesByFilterUseCase a través de la inyección de dependencias.
    - PARÁMETRO: Enviar el SaleFilter inicializado al método correspondiente de la interfaz.
    - RETORNO: Recibir un IAsyncEnumerable<Sale>.
    - VISUALIZACIÓN: Mostrar los resultados en formato tabular incluyendo: Folio, Fecha, Monto Total.
    - MANEJO DE ERRORES: Si no hay ventas en el rango, mostrar un mensaje amigable "No se encontraron ventas en el rango de fechas especificado".
  </rule_set>
  
  <rule_set name="Formato Tabular de Resultados">
    - ENCABEZADOS: Mostrar columnas claramente diferenciadas con bordes o separadores (Folio | Fecha | Monto Total).
    - ALINEACIÓN: Usar alineación izquierda para texto (Folio), centrada para Fecha, y derecha para Monto Total (numérico).
    - FORMATO DE FECHA: Mostrar en formato legible (dd/MM/yyyy HH:mm:ss).
    - FORMATO DE MONTO: Mostrar con símbolo de moneda si está disponible en la entidad (ej. $1,234.50).
    - TOTALES AGREGADOS: Opcional pero recomendado: mostrar al final el "Total de Ventas en el Período".
  </rule_set>
  
  <rule_set name="C# 14 y Limpieza">
    - Utilizar 'Raw String Literals' (triple comillas) para la visualización tabular expandida.
    - Inyectar IFetchSalesByFilterUseCase a través de un 'IServiceScope' manual dentro de la opción de menú.
    - Si la lógica de validación de fechas o visualización tabular es extensa, puede reutilizarse dentro de un método local o extraerse a un método helper.
    - El código debe permanecer asegurando que Program.cs siga siendo legible.
  </rule_set>

  <rule_set name="Preservación de Funcionalidad">

    - NO ROMPER: El flujo de control del programa actual ni la estructura de scopes.

  </rule_set>
